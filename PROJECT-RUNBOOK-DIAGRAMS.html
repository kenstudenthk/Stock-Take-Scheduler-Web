⚙ Schedule Generation Algorithm

Internal logic of utils/scheduleGeneration.ts
flowchart TD
    A([generateSchedule]) --> B["Input: pool, startDate, shopsPerDay, groupsPerDay"]

    B --> C[currentDate = getNextWorkingDay]
    C --> D{"shopIndex less than pool.length?"}

    D -->|No| E([Return scheduled array])

    D -->|Yes| F[Slice next N shops]
    F --> G[Assign each shop]
    G --> G1["scheduledDate = currentDate"]
    G --> G2["groupId = idx mod groupsPerDay + 1"]

    G1 & G2 --> HH[shopIndex += shopsPerDay]
    HH --> II[currentDate = next day]

    II --> JJ{isWorkingDay?}
    JJ -->|No Weekend| KK[Skip to next day]
    JJ -->|No Holiday| KK
    KK --> JJ
    JJ -->|Yes| D

⚡ Batch Operations & Retry Logic

How utils/batchOperations.ts handles concurrent API calls
flowchart TD
    A([executeBatch]) --> B["Split items into chunks of 5"]

    B --> C[Process chunk in parallel]
    C --> D{Each item}

    D --> EE[Attempt operation]
    EE --> FF{Success?}
    FF -->|Yes| GG[Add to successful list]

    FF -->|No| HHH{4xx error?}
    HHH -->|Yes Client error| III[Add to failed list, no retry]

    HHH -->|No Server error| JJJ{Retries left?}
    JJJ -->|Yes| KKK["Wait with exponential backoff"]
    KKK --> EE
    JJJ -->|No| III

    GG & III --> LL{More chunks?}
    LL -->|Yes| MM["Wait 100ms between batches"]
    MM --> C
    LL -->|No| NN[Return BatchResult]

    NN --> OO{Any failures?}
    OO -->|Yes| PP[retryFailed, one more attempt]
    OO -->|No| QQ([All done])
    PP --> QQ

    style III fill:#ef4444,color:#fff,stroke:none
