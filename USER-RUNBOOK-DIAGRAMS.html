🔒 Login & Authentication Flow

How users authenticate and what happens on success or failure
flowchart TD
    A([Open App]) --> B{First Visit?}
    B -->|Yes| C[PWA Install Guide]
    C --> D{Install or Skip?}
    D -->|Install| E[Install PWA]
    D -->|Skip| F[Continue in Browser]
    E --> F
    B -->|No| F

    F --> G[Login Page]
    G --> H{Have Account?}
    H -->|No| I[Register Tab]
    I --> J[Fill Name + Email + Password]
    J --> K[Account Created as User Role]
    K --> G

    H -->|Yes| L[Enter Alias Email + Password]
    L --> M{Token Valid?}
    M -->|No| N[Red Connection Indicator]
    N --> O[Go to Settings]
    O --> P[Paste New Token]
    P --> M

    M -->|Yes| Q{Credentials OK?}
    Q -->|No| R[Error Message]
    R --> L
    Q -->|Yes| S{Mobile Screen?}
    S -->|"Yes, 640px or less"| T[Mobile Map View]
    S -->|No| U[Dashboard]

📅 Schedule Generation Workflow

End-to-end flow from configuration to SharePoint sync
flowchart TD
    A([Generator Page]) --> B{Which Tab?}

    B -->|Regular Schedule| C[Step 1: Configure]
    B -->|Reschedule Pool| D[Pool Tab]

    C --> C1[Select Regions]
    C --> C2[Select Districts]
    C --> C3[MTR Toggle]
    C --> C4[Set Start Date]
    C --> C5[Shops/Day + Groups/Day]

    C1 & C2 & C3 & C4 & C5 --> E[Click Generate]

    E --> F{Unplanned Shops Found?}
    F -->|No| G[No shops match filters]
    G --> C
    F -->|Yes| H[Step 2: Preview Table]

    H --> I[Review: Date + Group + Shop]
    I --> J{Approve?}
    J -->|No / Edit| C
    J -->|Yes| K[Step 3: Confirm & Sync]

    K --> L[Batch Update to SharePoint]
    L --> M[Progress Bar]
    M --> N{All Succeeded?}
    N -->|Yes| O([Schedule Saved])
    N -->|Partial| P[Retry Failed Items]
    P --> L

    D --> D1[Load Pool Shops]
    D1 --> D2[Same Configure Flow]
    D2 --> E

⚡ Dashboard Shop Actions

Decision tree for shop management actions from the Dashboard
flowchart TD
    A([Dashboard]) --> B[Click Shop Card]
    B --> C{Choose Action}

    C -->|Reschedule| D[Smart Reschedule Modal]
    D --> D1[Pick New Date]
    D --> D2[Select Group]
    D1 & D2 --> D3{Confirm?}
    D3 -->|Yes| D4[Update SharePoint]
    D3 -->|Move to Pool| E1

    C -->|Move to Pool| E1[Clear Date + Set Rescheduled]
    E1 --> E2[Shop enters Reschedule Pool]
    E2 --> E3[Generate Pool Schedule later]

    C -->|Close| F1[Confirm Close]
    F1 --> F2["Status: Closed"]
    F2 --> F3[Removed from active scheduling]

    C -->|Resume| G1["Status: Pending"]
    G1 --> G2[Shop re-enters active pool]

    D4 --> H([Refresh Data])
    E2 --> H
    F2 --> H
    G1 --> H

🔑 Token Refresh & Auto-Sync

How the Graph API token is refreshed and shared across all users
flowchart LR
    subgraph UserA["User A"]
      A1[See Truck Banner] --> A2[Open Graph Explorer]
      A2 --> A3[Sign In + Copy Token]
      A3 --> A4[Paste in Settings]
    end

    A4 --> B[(localStorage)]
    A4 --> C[(Cloudflare KV)]

    subgraph AutoSync["Auto-Sync"]
      C --> D{Every 5 min poll}
      D --> E[User B fetches token]
      D --> F[User C fetches token]
      D --> G[User D fetches token]
    end

    E --> E1[localStorage updated]
    F --> F1[localStorage updated]
    G --> G1[localStorage updated]

    E1 & F1 & G1 --> H([All sessions refreshed])

    D --> E[Tap GPS Button]
