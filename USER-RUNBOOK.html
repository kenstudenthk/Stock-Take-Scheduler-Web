01

Getting Started

What is this app?

Stock Take Scheduler Web helps you plan and manage inventory stock takes across all HKT/PCCW retail shops in Hong Kong and Macau. You can:

  • Generate optimized schedules that avoid weekends and public holidays
  • Track which shops are planned, completed, or need rescheduling
  • View shops on a map with route navigation
  • Track inventory assets per shop
  • Export schedules to Excel or PDF

How to access

  1. Open the app URL in your browser (Chrome or Edge recommended)
  2. If you're not using the installed app yet, you'll see a full-screen Install Guide with tabs for iPhone/iPad, Android, and Desktop — each with numbered steps for your device. Tap “Continue in Browser →” to skip installing. This guide is remembered once dismissed and won't reappear on that device.
  3. You'll land on the Login page

User roles

RoleWhat you can do
AdminEverything — generate schedules, manage shops, manage users, manage inventory
App OwnerSame as Admin
UserView dashboard only

Login & Account

Logging in

  1. Enter your Alias Email (company email)
  2. Enter your Password
  3. Click Login

First-time setup

If you don't have an account:

  1. Click the Register tab
  2. Fill in your Name, Company Email, Alias Email, and Password
  3. Click Register
  4. Your account is created with User role. Ask an Admin to upgrade it if you need more access.

Changing your password

  1. On the Login page, use the Change Password option
  2. Enter your Alias Email, old password, and new password

Connection issues at login

If you see a red connection indicator, the SharePoint token may be expired. Click “Go to Settings” to update it — see Section 11.

Dashboard

Your home page — a bird's-eye view of all stock take operations.

Statistics cards

04

Schedule Generator

Required permissiongenerate_schedule — Admin / App Owner

Creates optimized stock take schedules using a 3-step wizard.

Step 1 — Configure

  1. Select Regions — HK Island, Kowloon, N.T., Lantau, Macau. Leave empty for all.
  2. Select Districts — optionally narrow down further
  3. Include MTR Shops — toggle MTR-located shops in or out
  4. Start Date — when scheduling should begin
  5. Shops Per Day — default 9
  6. Groups Per Day — default 3, labeled A/B/C

The stats banner at the top shows how many unplanned shops match your filters.

Step 2 — Preview

After clicking Generate, a preview table shows date, group, shop name, brand, and district for every shop. The algorithm automatically skips weekends and Hong Kong public holidays, and groups shops geographically so each group covers nearby shops. Review carefully before proceeding.

Step 3 — Sync to SharePoint

Click Confirm & Sync to save. You'll see a progress bar, a success/failure summary, and the option to retry any failed items.

Reschedule Pool tab

Switch to the Reschedule Pool tab to generate schedules specifically for shops previously moved to the pool — same 3-step wizard.

Reset functions

Required permissionreset_schedule — Admin / App Owner
  • Reset by Date Range — clear schedules within a specific range
  • Reset All — clear all planned schedules (requires confirmation)

Settings & Token Management

Why tokens?

The app connects to SharePoint through Microsoft's Graph API, which requires an access token valid for roughly 60 minutes. When it expires: a red animated banner reads “TOKEN EXPIRED: UPDATE IN SETTINGS”, data stops loading, and any save/sync operation will fail.

How to refresh the token

  1. Go to Microsoft Graph Explorer
  2. Sign in with your Microsoft account (must have SharePoint access)
  3. Confirm you have the Sites.ReadWrite.All permission
  4. Open the Access Token tab and copy the entire token
  5. Open the app's Settings page, paste the token, click Save
Token auto-syncOnce saved, the token syncs automatically to every other open session via Cloudflare — everyone picks it up within 5 minutes. You don't need to update it yourself if a teammate already did.

Connection status

The Settings page shows a status dot: Green — connected; Red — token expired or connection failed.

12

Mobile Map Navigation (Field Engineers)

On screens ≤640px, you're taken to the Mobile Map view automatically after login — built for navigating between shops during stock takes.

Features

  1. Date Picker — view shops scheduled for any date, not just today. Defaults to today.
  2. Group Selector — switch between Group A, B, or C
  3. Shop List — sorted by distance, filtered to the selected date + group
  4. GPS Location — on-demand button (saves battery); your position shows as a blue pulsing dot
  5. Distance — straight-line distance to each shop

Route navigation

Tap a shop, then tap the floating Route button to get directions:

  • Walking Route — shown if the shop is within 1km, with turn-by-turn directions and estimated time
  • Transit Route — bus/MTR directions with route numbers, via stops, transfer points, and MTR entrance/exit info
If a route can't be foundThe panel shows a “Route unavailable” message instead of staying blank — try again once you have a solid GPS fix.

Tips

  • Tap any route segment header to zoom the map to that segment
  • The bottom sheet is collapsible — drag the handle to expand or minimize
  • GPS only activates when you press the button — it doesn't track you continuously
  • Use the date picker to check tomorrow's route before you head out

FAQ & Troubleshooting

The app shows “TOKEN EXPIRED” — what do I do?

Go to Settings and paste a fresh token from Graph Explorer. Or wait — if a teammate updates the token first, your app auto-syncs within 5 minutes.

I see no data after logging in

Check the connection status in Settings. If it's red, update the token, then click “Refresh Data”.

The map isn't showing

The map requires an AMap API key. If the map area is blank, contact your Admin to verify the AMap configuration.

I generated a schedule but don't see it in the Calendar

Click “Refresh Data” — the Calendar reads from the same data source and should show it after refresh.

A shop was scheduled on a public holiday

Check whether the holiday is in the system's list (covers 2025–2028). If it's newly announced, ask the development team to add it.

I accidentally closed a shop

An Admin or App Owner can click the shop card on the Dashboard and choose Resume.

I can't see certain menu items

Menu items are role-based. A “User” role only sees the Dashboard — ask an Admin to upgrade your role for more access.

The schedule generation seems wrong

Double-check your filters: are you targeting the right regions/districts, is “Include MTR Shops” set correctly, and is the start date right?

How do I export the schedule?

Go to Calendar → use Export to Excel or Export to PDF. Both let you choose a custom date range.

The app feels slow

Make sure you're on a stable network, try “Refresh Data” in case cached data is stale, and on the map view keep the tab active only when you need it — rendering can be intensive.

I submitted a bug report but nothing happened

Reports save straight to SharePoint and are reviewed periodically. For urgent issues, contact your Admin directly.

Can I use this app on my phone?

Yes — it's a PWA. If not installed yet, you'll see a full-screen install guide with device-specific steps (iPhone/iPad, Android, Desktop), or tap “Continue in Browser →” to skip it. Once installed it behaves like a native app, and on mobile you get the dedicated Mobile Map view with a date picker for checking any day's route in advance.

Multiple people updated the same shop — whose change wins?

Last write wins — there's no conflict detection. If two people edit the same shop at once, the most recent save overwrites the earlier one. Coordinate with your team to avoid clashes.
